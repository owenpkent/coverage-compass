# Getting help

This page tells you where to go depending on what you need.

## If you need help with your own Medicaid case, read this first

**Coverage Compass is a software project. It is not a legal-help line, and the
people who build it are not your lawyers or your advocates.** Posting your
Medicaid problem here will not get you legal help, and this is a public space,
so please do not share personal or case details on GitHub.

For real help with a Medicaid case in Colorado (appeals, individual advocacy,
work-requirement notices, HCBS waivers, the Buy-In program, or civil rights
legal support), contact the Colorado Cross-Disability Coalition (CCDC):

- **Phone: (303) 839-3056**
- **Web: [ccdconline.org](https://ccdconline.org/)**

CCDC has advocates who can actually help with your case. Coverage Compass
cannot.

If you are in immediate crisis or facing a same-day deadline, call CCDC by
phone. Do not wait on a web form or a GitHub post.

## If you have a question about the software

For how the app works, ideas, feedback, or general questions, use GitHub
Discussions:

- **[Discussions](https://github.com/owenpkent/coverage-compass/discussions)**

Discussions is the right place for open-ended questions. It keeps answers in
public where the next person can find them.

## If you found a bug or want to request something

Open an issue using one of the templates:

- **[Open an issue](https://github.com/owenpkent/coverage-compass/issues/new/choose)**

The templates cover bug reports, plain-language feedback, accessibility
problems, translation review, and rule or policy corrections. Pick the one that
fits. If you are not sure, start a Discussion instead.

Please do not put any personal or Medicaid case information in an issue. Issues
are public.

## If you found a security or privacy vulnerability

**Do not open a public issue for security or privacy problems.** This includes
anything that could send a user's documents off their device, leak information,
or weaken the privacy guarantees.

Follow the responsible-disclosure process in
**[SECURITY.md](SECURITY.md)**, which uses GitHub's private vulnerability
reporting so your report stays confidential.
