# Changelog

All notable changes to this project are recorded here. The project is in pre-v0.1 scoping, so entries describe documentation and outreach work rather than shipped features. Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). Dates are local Mountain time.

## [Unreleased]

### Added
- **The form-fill engine is ported: Coverage Compass can now fill a real official form.** The proven CDASS Enroll write side is in `web/src/lib` as a headless TypeScript module: tolerant pdf-lib helpers (`fill/util.ts`; a renamed field degrades to a logged warning, never a crash), the profile schema (`profile/schema.ts`, CDASS field set; Medicaid archive sections still to come with CCDC), and the per-form mappings for the CDASS/PPL Attendant Enrollment Packet 2026 with its embedded I-9 (`fill/forms/`). The blank official template ships at `web/public/forms/` and the first document-library registry entry lands at `rules/co/forms/cdass-attendant-packet-2026.yaml` (status: draft, pending CCDC advocate review). The exact-copy regression test (`fill/packet2026.test.ts`) fills the real 22-page template with fictional data and proves the output keeps every page and every live field, values land where the mapping says, the under-18 attestation stays unchecked for an adult date of birth, and the paper-check path works. All pdf-lib imports in the fill layer are dynamic, so the lazy-chunk discipline holds. Engine invariants carried over: never flatten, never fabricate signatures. Suite is now 72 tests. Remaining for user-facing fill: the Medicaid schema sections, capture reuse, the review-and-generate UI, and the IndexedDB archive (see `docs/form-fill-engine.md`).
- **Test coverage grew from 39 to 70 unit tests** across five new suites. `exampleLetters.test.ts` pins each demo letter to the classification and deadline behavior its button promises (so a rules or wording change that breaks the demo fails loudly), and checks every letter declares itself fictional in its first line. `legal.test.ts` guards full English/Spanish parity across the Terms and Privacy documents, the presence of the "fullest extent permitted by law" waiver clause in both languages, the as-is and $0-cap clauses, and CCDC phone-number consistency. `Triage.test.tsx` covers the paste flow end to end, the example-letter flow with reset, and the too-short-paste edge. `format.test.ts` covers local-date parsing (no UTC day-shift), bilingual long dates, and the urgency bands including past-due. `plainLanguage.test.ts` covers locale fallback and the translation-pending flag.
- `npm test` now runs the suite once (CI-safe); watch mode moved to `npm run test:watch`. Previously `npm test` started vitest in watch mode and hung any non-interactive caller. `web/README.md`, `CONTRIBUTING.md`, and the PR template updated to match.
- **Light and dark mode.** The palette still follows the system preference by default, and a new header toggle (`ThemeToggle.tsx`) pins light or dark for the session via `<html data-theme>`. Session-only, like the language toggle: no cookie, no localStorage. `color-scheme` is set so native controls match.
- **Terms of Use and Privacy Notice** (`content/legal.ts`, `LegalPage.tsx`): bilingual in-app pages reachable at `#terms` and `#privacy` from the footer and from the consent gate, readable *before* accepting the release. The Terms carry as-is/no-warranty language, assumption of risk, a waiver and release "to the fullest extent permitted by law" (covering information errors, missed deadlines, and privacy/security incidents), a $0 liability cap, and Colorado governing law. The Privacy Notice is honest about the local-only design, what the host sees, and what no one can guarantee. Both pages display a "draft, not yet reviewed by a lawyer" banner; CCDC counsel review is required before this is treated as more than a demo. The consent gate now states that continuing accepts both documents, and the page title tracks the open page (WCAG 2.4.2).
- **Live demo deployment** at [coverage-compass-6ky.pages.dev](https://coverage-compass-6ky.pages.dev) (Cloudflare Pages). The build's `_headers` file is honored, so the full Content-Security-Policy (including `frame-ancestors 'none'`), `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, and `Referrer-Policy: no-referrer` are sent as real response headers.
- **Consent gate (`ConsentGate.tsx`):** a bilingual click-through release shown before the tool can be used. Plain-language points: everything stays on-device, demo status, draft content, not legal advice, the user chooses what to open on their screen, no warranty. Acceptance is session-only React state (no cookie, no localStorage), so the gate shows every visit, consistent with "nothing persists by default". Wording is flagged as pending CCDC legal review. On accept, focus moves to the revealed main content. Axe-tested in both states.
- **Example letters (`fixtures/exampleLetters.ts`):** four clearly-fictional sample letters (procedural termination, renewal packet in Spanish, work-reporting notice, exemption denial) reachable from a "Try an example" row. Dates are generated relative to today so the deadline demo never goes stale; every letter's first line says it is a sample; buttons feed the normal classify path.
- **Deadline extractor recall:** bare "by", "para el", and "hasta el" now count as deadline triggers when (and only when) the date starts immediately after them, so "return the packet by June 30, 2026" is read while "reviewed by John Smith on 06/01/2026" stays ignored. Five new unit tests.
- **Demo badge** in the header, and `docs/document-library.md`: the v0.2 design for a curated registry of verified fillable forms (per-form YAML in `rules/co/forms/`, exact-copy smoke tests, advocate review loop) plus a best-effort path for user-uploaded AcroForm PDFs.
- **v0.1 read side (notice triage), implemented end to end in `web/`.** A dropped PDF or photo, or pasted text, is read entirely on the device and classified against the Colorado rule library, then shown as a plain-language summary with the deadline (date and days remaining), what happens if you do nothing, and one to three concrete next actions, in English or Spanish. New modules: `lib/pdf.ts` (pdf.js text extraction, worker bundled from our own origin), `lib/ocr.ts` (tesseract.js with vendored worker, WASM cores, and language data), `lib/deadline.ts` (bilingual date extraction that only trusts a date next to a deadline word), `lib/rules.ts` + `lib/plainLanguage.ts` (deterministic classifier over the YAML rules), `lib/summaryPdf.ts` (downloadable one-page summary via pdf-lib), and the `DeadlineCard`, `NextActions`, `LetterSummary`, `LanguageToggle`, and `Triage` components.
- Build-time rule codegen (`web/scripts/gen-rules.mjs`): compiles `rules/co/letter-types.yaml` into a typed `web/src/lib/rules.generated.ts`, keeping the YAML the advocate-editable source of truth while the app imports typed data.
- OCR asset vendoring (`web/scripts/vendor-ocr.mjs`): copies the tesseract worker, WASM cores, and English/Spanish language models from `node_modules` into `web/public/vendor` (gitignored, reproducible from the lockfile), so the photo path makes no third-party network request.
- Internationalization with react-intl: English and Spanish message catalogs (`web/src/i18n/`), an in-memory language toggle, and a draft Spanish layer in the rule library marked pending native-speaker and CCDC review.
- PWA: web manifest, a service worker that precaches the app shell and runtime-caches the OCR assets, and SVG app icons, so PDF and paste-text triage work offline after first load.
- Production Content-Security-Policy injected at build time (`connect-src 'self'` blocks data exfiltration; the vendored OCR/PDF assets stay same-origin), and an ESLint 9 flat config that actually runs the `jsx-a11y` accessibility rules (there was previously no eslint config, so lint never ran).
- Tests: unit tests for the deadline extractor and the classifier, an i18n catalog-parity test, a CCDC-phone-consistency guard over the rules, and component plus axe-core accessibility tests for the landing page and the result view.
- Hardening from a multi-agent review of the read side (each finding adversarially verified before fixing). Notable: forced `cacheMethod: "none"` so tesseract never persists its language model to IndexedDB (honoring "nothing persists by default"); the classifier now breaks score ties toward the higher-stakes letter type (an urgent termination is never hidden behind a benign renewal) and no longer double-counts overlapping patterns into a falsely high confidence; the deadline extractor no longer mistakes an enrollment/effective date for an action deadline; OCR progress no longer floods the screen-reader live region; focus returns to the upload view on reset/error; the printable PDF carries the "unreviewed Spanish" caveat; HEIC photos get specific iPhone guidance instead of a generic error; and the build emits a `_headers` file (full CSP plus `frame-ancestors 'none'` and clickjacking headers) for header-capable hosts.
- README status badges (license, PRs welcome, good first issues, accessibility, privacy, status) via shields.io.
- `web/src/lib/fill/fillForm.ts` and a Vitest exact-copy test: pdf-lib adopted (lazy-imported) as the write-side groundwork from the proven engine, the first step of the form-fill adoption plan (good first issue #1).
- `.github/CODEOWNERS`: default review owner (@owenpkent), with a clearly-marked placeholder for adding CCDC advocate review of `rules/**` and user-facing copy once a reviewer has a GitHub handle.
- `SUPPORT.md`: where to get help; routes Medicaid case help to CCDC (303-839-1775) and software questions to Discussions, and points security reports to SECURITY.md.

Five archetypes for the v0.1 notice-triage flow (`01-single-screen`, `02-linear-wizard`, `03-conversational`, `04-two-pane`, `05-print-first`) and five for the v0.3 appeal workflow (`a01-progress-tracker`, `a02-dashboard`, `a03-advocate-conversation`, `a04-two-pane`, `a05-document-first`). `index.html` and `appeals-index.html` link them with one-line trade-offs. `shared.css` encodes WCAG 2.2 AA tokens, dark / reduced-motion / high-contrast support, a 48px tap-target floor, and an appeals-specific dual-clock urgency banner that surfaces continuation-of-benefits and appeal-deadline clocks together.
