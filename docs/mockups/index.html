CC Coverage Compass UI brainstorm, May 2026

UI brainstorms

Two brainstorms so far. Below are five takes on the v0.1 notice triage flow. For the v0.3 appeal workflow brainstorm, see appeals-index.html.

These five are about the same v0.1 letter triage flow: same letter, same plain-language output, same trust posture. The only difference is the shape of the interaction. Open each one and pretend you just got the letter.

All five share shared.css so the visual language stays consistent. The shared sheet aims at WCAG 2.2 AA with AAA contrast on body text, respects prefers-reduced-motion, prefers-color-scheme, and prefers-contrast, keeps tap targets at 48px or more, and uses semantic HTML.

Mockups

1

Single screen drop target

The whole viewport is a drop zone. Result renders below in place.

Lowest cognitive load. Hardest to surface secondary inputs.

 2

Linear wizard

One question per step. Big buttons. Clear progress.

Excellent for screen readers and stressed users. Feels slow for repeat use.

 3

Conversational

Short interview with large choice buttons, not a freeform chat.

Lowers fear, sets up v0.3 appeals nicely. Live-region churn is risky.

 4

Letter and translation, side by side

Original letter on the left, plain-language meaning on the right.

Best for advocate-and-member pairs. Letter image itself is not accessible content.

 5

Print-first one-pager

Output looks like a printable document from the start. Save, print, hand to family.

Matches paper-based mental model. Large print and sharing come for free.

How to evaluate these

  • Open each mockup in a new tab. Try them with the keyboard only (Tab, Shift+Tab, Enter, Space).
  • Switch your OS to dark mode and reload. All five should remain readable.
  • Turn on "reduce motion" in your OS. Nothing should animate.
  • Zoom to 200% in the browser. Layout should not break.
  • Run a screen reader (NVDA, VoiceOver, or TalkBack) for a minute on each.
